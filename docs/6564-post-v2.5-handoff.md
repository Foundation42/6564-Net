## 1. The Concurrency Realization: The Machine Never Had Preemption

**The insight.** Every context switch in this architecture is voluntary or fatal. Parks (`LSTN`, `YLD`, `RECV`, and `SEND` on a full SQ) are architecturally visible points; the only involuntary transition is a fault (crash, watchdog), and faults were never resumable — supervision restarts via `SPWN` with fresh registers. There is no moment where hardware must preserve register state on anyone's behalf.

**Decision (measure before adopting): collapse the banked register file entirely.**

- Registers (A, X, Y, SP, P) become **one shared set**, volatile across parks. The compiler/programmer treats every potentially-parking instruction as a full register clobber. The clobber set is exactly: `YLD`, `LSTN`, `RECV`, `SEND`. Enumerable, small, and `SEND` is on the list even though the v0.1 instant-drain RBC never parks it — the spec says it may.
- A context becomes: **near page + run-queue entry + control block**. Nothing else.
- The **control block** is the state that must survive parks and `SPWN`: incarnation, exit link, watchdog base budget, and the new declaration ceiling (§2). Decision: migrate it into an architected near-page stripe — with **hardware write-protection against the owning context** (an actor must not edit its own leash). Supervisor/privileged writes only. This stripe is the one new mechanism the collapse requires.
- `SPWN` simplifies: nothing to reset except the control-block updates it already does.
- The 6502-heritage argument, for the commit message: registers were never where state lived — variables lived in zero page and registers were transient scratch. Near-page indirect addressing means parked state is frequently *used in place*, never "restored."

**Implementation path.** This is measured *through joe* (§6): the compiler's park-point liveness analysis is the register allocator, so converting the demo corpus to joe and reading the frozen table IS the experiment. Do not hand-convert the assembly corpus first — that measures programmer heroics, not the convention.

**Pre-registered predictions (Claude's, on the record):** ring stays under 70 cy/pass; the spill tax is smaller than feared because idiomatic code keeps persistent state near-page-resident; the collapse pays for itself (silicon simplicity + SPWN simplification) before the pipeline demo finishes converting. Christian reserves judgment pending the numbers — that's the point.

## 2. Watchdog: `WDEX` — Bounded, Supervisor-Ceilinged Declarations

**Problem.** Some operations legitimately exceed the burst budget (Tier-1 vector blocks, serial grinds). A naive `WDF` "feed the dog" instruction is rejected: a hung actor spinning in a loop that contains the feed defeats the watchdog exactly where it's needed — the classic embedded failure.

**Decision.** The watchdog is the *supervisor's* contract about worst-case unyielding time, so extensions are supervisor-bounded:

- Control block gains a second privileged value: the **declaration ceiling** (alongside the base budget).
- **`WDEX #n`**: sets the current burst's remaining budget to `n`. Faults the executing context (`bad_descriptor`-class, new code `wdex_ceiling`) if `n` exceeds the ceiling. At the next park, everything resets to the base budget. One outstanding declaration at a time; a second `WDEX` in the same burst replaces (still ceiling-checked).
- Guarantee preserved: total unyielding time never exceeds what the supervisor authorized. Declarations are visible in the code and auditable in the trace.
- Design note to keep: watchdog pressure is a *nudge toward Tier 2* — anything long enough to want `WDEX` routinely probably wants to be a fabric SQE with the actor parked. `WDEX` exists for the honest residue.
- joe surface: `bounded N { … }` (already in the sketch, §2.5 / lowering table).

**Encoding:** wherever the opcode map has room; likely the extended page (§3). Null-cost rule applies.

## 3. Floating Point, Tier 0 and Tier 1

**Tier 0 — scalar, no new registers.** FP64 lives in A; FP32 widens on load. Scalar FP ops (`FADD`, `FSUB`, `FMUL`, `FDIV`, `FSQRT`, compares setting P flags, int↔float converts) are accumulator operations — the most 6502 shape available. Because Tier 0 adds no state, the volatility question doesn't even arise for it.

**Tier 1 — a small vector set, volatile by rule.** V registers (working proposal: V0–V7 × 512-bit; size to what the workloads justify, not more). **Volatile across parks — never banked, never saved by hardware.** Compiler treats them caller-saved at every clobber point (§1's set); live vector state spills to the near page. A watchdog trip mid-vector-block loses the state by design: trips are deaths, deaths restart.

**Encoding — the extended page.** One prefix byte (pick a free column) opens a second 256-entry opcode page for FP/vector ops. **Non-stacking, hard rule** — one prefix, one page, done. The Z80's DD/CB prefix soup is the cautionary tale; decode must stay countable and the assembler honest. The comptime ISA table grows a second page; duplicate detection extends across both.

**Numerics (binding for Tiers 0–1):** IEEE 754, fixed rounding mode (round-to-nearest-even), **no FTZ/DAZ, no silent fusion** (FMA is its own opcode or absent in v1). Bit-exact deterministic — this is inside the ISA, so it's inside the determinism bar (§5).

## 4. Tier 2: Registered Regions and Accelerator Actors

**The model.** Large operations (matmul, splat math, inference state pushes) are **fabric requests**: the core dispatches an SQE to an accelerator actor at mesh coordinates and parks or proceeds; the accelerator posts a completion; the actor's wake handles the result. A discrete TPU, a Vulkan compute path, or a software polyfill are indistinguishable per spec §7.5 — silicon is an optimization, never an interface.

**Registered regions — ownership for big buffers.** Composed entirely from existing mechanisms:

- A region is **(base, length, capability token, ownership bit)** — §6.4's token plus §6.2's ownership bit, widened from one SQE buffer to one region.
- **Grant-on-submit:** the SQE carries the region descriptor; acceptance sets the ownership bit — the region is hardware-owned.
- **Completion record = release fence.** The same one rule, unchanged.
- **Revocation = clear the token** (already immediate per §6.4). This is the fencing story DMA designs usually lack: revoke a wedged accelerator and its late DMA gets reject-completed instead of scribbling on reclaimed memory.
- **The polyfill path settles local-vs-remote:** a local accelerator DMAs the region; a remote/software implementation *pulls it through the network window with the same token* — which finally legitimizes remote reads (v0.1 made them fault precisely because there was no ownership story; the PTT `write` right was explicitly "reserved for future registered-region RDMA" — this is that future). Piecemeal push is not an alternative design; it is the polyfill implementation of the region design.
- **Streaming/zero-copy reuse:** a *ring of region descriptors* — accelerator owns region N while the core fills N+1. Double-buffering as ordinary queue-pair discipline. No new mechanism.
- Coherence stance inherited unchanged: no coherent shared mutable memory; ownership handoff with the completion as fence; implementations flush/invalidate at grant and release.
- joe surface: `region` declarations and `grant` (sketch §2.5); a granted region is **inaccessible by type** until the matching completion case rebinds it — §6.2 promoted to compile time.

**Sim work:** region table + token check on the device/DMA path; a matmul device on the peripheral row with two implementations (in-proc "TPU" and a fabric-remote polyfill) to prove indistinguishability; a saturation test in the flood regime (the deferred-grant lesson says validity contracts die at saturation first — go looking there).

## 5. Tiered Determinism

**The bar, decided:** **bit-exact deterministic within the ISA** — Tiers 0–1, per §3's numerics. Outside the ISA, honesty over pretense:

- Device contracts carry an explicit **`deterministic` flag**. Set: the contract specifies exact arithmetic *including reduction order* — polyfill-checkable, replay-safe across implementation substitution, slower on real GPUs. Clear: results are contract-accurate but implementation-varying; replay is guaranteed **per-federation** (same seeds, same devices → same bits) but not across device substitution.
- The application programmer decides whether it matters and picks mitigations (choose deterministic devices, verify against polyfill, tolerate the variance). The machine never pretends two accelerators agree when they don't.
- **Spec delta: a numeric addendum to §7.5.** Without it, "no client can tell" is false at the bits for Tier 2, and bit-identical replay — the crown jewel — gets quietly broken by the first parallel reduction.

## 6. joe

Decided and sketched — full detail in `joe-v1-sketch.md`, which is the authoritative companion to this brief. Summary for orientation:

- **Go's surface, Erlang's semantics, occam's discipline.** Lowercase joe, `.joe` files. Named for Armstrong (and, it turns out, Christian's first cat — the naming precedent being his first language, Matt, after his second).
- Constructs are the silicon: `actor`/`serve`/`case…where`/`after`/`spawn…restarts…watchdog`/`chain…on_break`/`bounded`/`quiesce`/`region`/`grant` — the lowering table in the sketch maps each to its ISA mechanism one-to-one.
- **`var` = near page (survives parks); `let` = registers (volatile).** The §1 convention as vocabulary.
- Negative space is the spec: no shared mutable state, no synchronous calls, no transport-ack visibility, no touching granted regions, no undeclared long compute.
- **Compiler:** Zig, same repo. Recursive descent → typed AST → park-point liveness → `.asm` text → existing asm.zig. No IR, no classical register allocator — park-point liveness with the near page as the surviving frame *is* codegen's hard part, which is why building joe is the §1 experiment.
- **joe also supplies the MAC re-trial corpus:** compiled code has real subroutine reuse; re-run the MAC measurement (deferred in `measurements.md`) once the demo corpus compiles.

## 7. Suggested Order of Work

1. **`WDEX` + control block ceiling** — smallest diff, needed by joe's `bounded`, null-cost checkable immediately.
2. **Extended opcode page + Tier 0 scalar FP** — ISA table growth, decode, assembler; deterministic numerics tests.
3. **joe v1 compiler** — parser, types, liveness, codegen to asm.zig. Convert pingpong first (smallest protocol), then the suite.
4. **Bank-collapse measurement** — flip the convention in compiled output, read the frozen table, ledger the verdict against §1's predictions. Adopt or retain banks accordingly.
5. **Tier 1 vectors** — only after 4, since the volatility convention should be settled before vector spill code exists.
6. **Registered regions + matmul device (both implementations) + saturation test.**
7. **MAC re-trial** on the compiled corpus; update `measurements.md`.
8. **Spec v2.6 draft**: §5.4 WDEX, §6.2 regions, §7.5 numeric addendum, FP tiers section, and — verdict permitting — the context collapse.

Throughout: predictions before measurements, failures in the ledger, frozen-table null-cost checks on every mechanism, deterministic replay tests extended to cover FP and regions.
